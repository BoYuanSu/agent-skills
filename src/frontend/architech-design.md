# 前端分層架構設計規範 (基於單向資料流)

本規範旨在定義前端複雜元件（如大型表格、數據看板等）的架構設計，透過明確的分層職責與單向資料流，達到程式碼解耦、易於維護與高效渲染之目的。

## 一、 參數與狀態控制層（State & Parameter Layer）
這一層負責收集使用者的操作與環境變數，將其轉化為向後端請求資料的依據，並監控需重新整理的狀態。
*   **輸入與狀態管理：** 讀取路由參數，並結合使用者選擇的篩選條件、分頁與預設設定，合併出元件的運作狀態。
*   **輸出準備：** 將上述狀態組裝成 API 所需的查詢參數（Payload）。
*   **狀態連動：** 偵測關鍵狀態或事件的變化，觸發重新獲取資料（Refetch），驅動整條資料流重新執行。

## 二、 資料請求層（Request Layer）
此層專注於與後端 API 的通訊，將參數層的指令發送出去並取回原始資料。
*   **執行請求：** 將組裝好的查詢參數交給資料請求模組（如 React Query/RTK Query 或自訂 Hooks）發起請求。
*   **控制與回傳：** 透過內部控制機制（如啟動條件 `enabled`）決定是否發送請求，並成功取回後端的原始資料回應（Raw Data）。

## 三、 資料正規化層（Normalization Layer）
**這是整個功能最重要的資料轉譯邊界**，負責將後端多變的 API 格式收斂、解耦，轉換成前端視圖可以直接消費的標準化結構。
*   **原始資料對齊：** 針對不同 API 格式的多樣性做相容性抽取，統一成單一的原始資料輸入。
*   **分流轉譯：** 將對齊後的原始資料進行格式轉換與清洗。
*   **標準化產出：** 依據功能需求，產出專責的畫面資料結構，例如：主列表資料、統計區資料、圖表所需之座標資料等，使呈現層無需處理複雜的欄位轉換邏輯。

## 四、 視圖呈現層（Presentation Layer）
此層完全不涉入 API 邏輯與複雜的資料轉譯，僅接收正規化層產出的標準化資料（與顯示開關設定），並依據職責拆分至各個子元件進行畫面渲染。
主容器元件接收資料後，會分發給以下子模組：
*   **配置層（如表頭/側邊欄）：** 呈現對應的結構，並處理操作事件（如排序、點擊），將事件回傳給容器層更新狀態。
*   **主內容呈現層（如表身）：** 負責逐列/逐格渲染，將標準化資料經過基礎轉換、格式化後，輸出為對應的 HTML 節點。
*   **輔助呈現層（如圖表）：** 將專屬的標準化數據對應成視覺元素（如 Canvas、SVG 或圖表元件）進行繪製。
*   **統計與加總層：** 接收統計資料，並依據設定決定是否呈現。

## 五、 架構總結
這四個分層完美體現了**「單向資料流（Unidirectional Data Flow）」**的設計概念。
從「狀態 -> 參數 -> 請求 -> 正規化 -> 渲染」，各層職責分離。設定面板的開關或顯示控制只會影響「視圖呈現層」，不會反向污染資料層，亦不會觸發不必要的 API 請求。

## 六、 針對簡單頁面的剪裁實踐 (KISS 原則)
當面對結構單純、互動較少的簡單頁面（如關於我們、簡單文章列表）時，強行套用完整的四層架構可能會導致過度設計（Over-engineering）。此時，應本著 KISS 原則對架構進行精簡與裁剪：

1. **省略「資料正規化層（Normalization Layer）」**
   *   **原因：** 簡單頁面的 API 通常已採用扁平化結構，且資料不需分流至多個複雜的視覺模組，無需額外建立資料轉譯邊界。
   *   **做法：** 請求取得的原始資料（Raw Data）可直接傳遞給呈現層進行渲染。

2. **合併「參數與狀態控制層」與「資料請求層」**
   *   **原因：** 簡單頁面通常無頻繁的狀態同步或複雜的查詢參數組裝。
   *   **做法：** 將這兩層合併為一個簡單的 API 呼叫 Hook（例如 `usePageData`），負責發起單一請求，保持與 UI 的解耦。

3. **保留「視圖呈現層（Presentation Layer）」但簡化元件結構**
   *   **原因：** 「接收資料並渲染畫面」是前端的最基本職責。
   *   **做法：** 雖然不需要拆分出繁複的子元件（如配置層、統計層），但**仍必須遵守 API 與 UI 分離的原則**。應該由專屬 Hook/容器層處理資料獲取，再將乾淨的資料傳給 UI 元件進行 HTML/JSX 渲染，避免將網路請求等資料邏輯直接寫入 UI 呈現元件中。

**裁剪後的簡化資料流：**
即使架構簡化，仍應維持 API（資料層）與 UI（呈現層）的清晰邊界：
```
API 呼叫 Hook (資料層) ──(傳遞資料)──> 呈現元件 (UI層)
```
